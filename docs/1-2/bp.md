# 程序设计基础

## 课程信息

本课程为大一计科大班的专业课（和 [SICP](../1-1/sicp.md) 为二选一的关系），在每年的上学期和下学期在鼓楼开设。

如果你是信计的，需要选修，需要在“跨专业课程”中选择；一般会有极少数人在大一上学期跨选这门课。

课程主要讲过程式程序设计，和一小部分计算机原理，语言方面选择了 C 与 C++ 相交的部分，主要面向零基础的学生。

这门课似乎在 MOOC 同步开发，链接：<https://www.icourse163.org/course/NJU-1468673161>

2026 春的分数构成为平时 10%（OJ 作业+图文作业+随堂讨论），上机 30%（5%+10%+15%），期中笔试 10%，
大作业 10%，期末笔试 40%。

你可以在我的大作业仓库找到今年的大作业：<https://github.com/nju-cs-2025-xmzheng/nju-bp-2026-spring>

## 课程评价

这门课给我的印象非常好，刘奇志老师人非常好，讲课也很有水平，整体课程难度也适合零基础的人学习。
至于我为什么会在大一下学期选这门课，首先我肯定不是主要来学知识的，主要是有一名大一的
[木斧助教](https://github.com/KEDIT2007)，那我肯定要支持一下对吧，大一的助教太罕见了（x）
好吧所以最终我选了这门课，发现刘老师讲的很好，我也学到了知识，算是意外之喜了；这应该是我为数不多全勤的专业课之一。

今年（2026）下学期开的时候情况比较奇怪，在一个特别大的教室上课，但是只有十几个人；课程开始时问了一下是否都有基础，
随后结果就是课程进度远快于秋季学期，不过后来也因为一些原因放慢了。由于课程进度不一样，且这是很小的班，
所以我这里写的一些情况可能会和大班不太相同。这个学期的期中期末都进行了一部分程度的创新，删除了概念题，
改为了较为特别的简答题，答案是开放的，比如：

> 针对计算机程序中的变量，你可以提 2 个你认为比较重要的问题吗？要求写出问题，并给出相应的回答。

> 如果要编写一个可维护性强的 C 程序，需要遵循一些编码规范。以下是两款 AI 给出的各自看重的五条规范，
> 以及它们的重要性。请你参照 AI 给出的规范，给出你最看重的一条规范，并结合切身体会解释原因。

所以这也导致拿高分的难度大大减小，笔试不再会因为糟糕的概念题卡掉很多极高分，本人没怎么认真学但也差点理论了。

平时的图文作业我最开始不知道很多题是选做的，后来期末才得知做选做的题会得到一些积分，可以收获小礼品（？
也算是误打误撞拿到了，真的很惊喜；每周实验课一般是发 OJ 作业，然后你可以写，不要求课上写完，也有可能拿来机试，
或者笔试，或者验收大作业。机试难度不大，和作业差不多，有人不到十分钟就写完了。笔试其实也不难，主要是细心就行，
题量不大，都是课上的知识，考前把 ppt 过一遍就很稳了。大作业的话，2025 秋是要求做一个沙威玛游戏，
不要求实现 GUI，这个应该是常态；最后统一验收，只验收一次应该。我这个学期的大作业的题目可以在我的
[仓库](https://github.com/nju-cs-2025-xmzheng/nju-bp-2026-spring/blob/main/docs/problem.md)
里找到，这里每完成一个部分都会需要提交一次，并且有可能会打回，最后全部做完以后要录一个视频，
然后线下验收会有助教现场要求在你的代码里实现新的功能，或者进行重构（这两个方向是自选的，取决于选了什么拓展实现），
可以详细描述出怎么改，不用完全改完，也可以使用 AI 完全写对；所以可以看到这学期助教工作量在验收方面非常大，
并且这整个大作业都是木斧助教一人设计的，可以说设计的很好，需要你在完成的过程中按照指引，多次重构自己的代码；
虽然允许使用 AI，但我还是尝试了一下完全手搓，我感觉最后对我也有收获。

今年（2026 春）期末卷最高分 99，均分 78.5，中位数 83.5；课程总评最高分99，均分、中位数高于期末卷，
总评最高分、均分都跟上学期一样（原谅我这里没找到数据了）

我觉得给分挺好的，课也挺好的。
